Panorama

Panorama is my tool for monitoring and analysis of performance issues of Oracle databases.
  • it focuses on usage by DBAs and also by software developers
  • provides easy access to several internal information
  • Offers support for:
    • Performance-optimization of applications
    • Analysis of runtimes and wait events
    • Preparation of Oracle database's internal information for easy usage
  • aims to issues that are inadequately analyzed and presented by other existing tools such as Enterprise Manager
  • immmediately startable as Java war-File with built-in Winstone application Server. ( java -jar Panorama.war )
  • may be deployed as web application to every JEE or web container (Glassfish, JBoss, Tomcat ...)
  • some functions of panorama requires licensing of "Diagnostic Pack" for connected Oracle-DB !
  • Oracle-User used for login with Panorama requires the right "SELECT ANY DICTIONARY"
  • if using tnsnames.ora it's directory should be targeted by ORACLE_HOME/network/admin or TNS_ADMIN environment variable
  • Panorama only reads via SELECT SQLs from database. No write access, structure change, compile etc.
  • Supports Oracle database on standard hardware as well as RAC-systems and engineered systems (EXADATA)

Panorama.war can be run as web archive in any application server or web container like Tomcat, Glassfish etc. .
For immediate usage Panorama.war contains the embedded application server "Winstone".
To run Panorama.war with Winstone simply type "java -jar Panorama.war". This starts the embedded application server within one or two minutes.
You can reach than Panorama's login screen in your browser via "http://localhost:8080/Panorama".

You can specify the following additional Java-options executing "java <options> -jar Panorama.war":
  • "-Xmx1024m" :maximum allowed memory allocation for JVM, minimum value of 1024 MB suggested for processing of large results
  • "XX:MaxPermSize=512M" : object space for java classes, suggested value 512MB
  • "-Djava.io.tmpdir=mydir" : define working directory for exraction of jar file, default is system's temp dir
  • "-Djruby.compile.invokedynamic=true" : Fix possible problems with some JREs resulting in "java.lang.ClassNotFoundException:"

You can specify additional Winstone-options executing "java -jar Panorama.war <options>":
  • Allow only SSL encrypted https-connections with disabled http-port:
    java -jar Panorama.war --httpPort=-1 --httpsPort=8443 --httpsKeyStore=<keystore file> --httpsKeyStorePassword=<keystore password>
  • Use another application prefix to access via "http://my_host:8080/MyPrefix": java -jar Panorama.war --prefix=/MyPrefix
  • All command line options of embedded winstone server are described here: http://winstone.sourceforge.net
