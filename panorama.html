Panorama

Panorama is my tool for monitoring and analysis of performance issues of Oracle databases.
  • it focuses on usage by DBAs and also by software developers
  • provides easy access to several internal information
  • Offers support for:
    • Performance-optimization of applications
    • Analysis of runtimes and wait events
    • Preparation of Oracle database's internal information for easy usage
  • aims to issues that are inadequately analyzed and presented by other existing tools such as Enterprise Manager
  • immmediately startable as Java war-File with built-in Jetty application Server. ( java -jar Panorama.war )
  • may be deployed as web application to every JEE or web container (Glassfish, JBoss, Tomcat ...)
  • some functions of panorama requires licensing of "Diagnostic Pack" for connected Oracle-DB !
  • Oracle-User used for login with Panorama requires the right "SELECT ANY DICTIONARY"
  • Panorama only reads via SELECT SQLs from database. No write access, structure change, compile etc.

Panorama is an Ruby on Rails application
Source code is available at http://github.com/rammpeter/Panorama

You can download Panorama as an immediately startable web archive (Panorama.war).

Panorama.war can be run as web archive in any appication server or web container like Tomcat, Glassfish etc. .
For immediate usage Panorama.war contains the embedded apllication server "Jetty".
To run Panorama.war with Jetty simply type "java -jar Panorama.war". This starts the embedded application server within one or two minutes.
You can reach than Panorama's login screen in your browser via "http://localhost:8080/Panorama".

With Jetty you can specify the following additional options executing "java ..options.. -jar Panorama.war":
  • "-Xmx1024m" :maximum allowed memory allocation for JVM, minimum value of 1024 MB suggested for processing of large results
  • "XX:MaxPermSize=512M" : object space for java classes, suggested value 512MB
  • "-Djetty.port=8090" : define port other than 8080 f√ºr Panorama's URL
  • "-Djava.io.tmpdir=mydir" : define working directory for exraction of jar file, default is system's temp dir
  • "-Djruby.compile.invokedynamic=true" : Fix problems with some JREs resulting in "java.lang.ClassNotFoundException: org.jruby.ext.krypt.asn1.RubyAsn1"
